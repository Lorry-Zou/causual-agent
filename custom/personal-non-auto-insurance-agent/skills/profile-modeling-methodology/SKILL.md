# 保险行业用户画像建模方法论

## Purpose
为profile-modeler提供系统化的用户画像建模框架，确保标签体系全面、可追溯且支持下游Agent（gap-diagnosis-engine、churn-recovery-agent）精准消费。

## When to Use
- 构建或更新C端用户全维度画像时
- 设计标签分类体系与元数据标准时
- 需要将用户行为数据转化为可操作的营销/服务洞察时
- 下游Agent请求特定标签数据时

## Core Knowledge

### 1. 保险场景RFM模型适配

传统RFM（Recency/Frequency/Monetary）在保险场景需做以下适配：

| 维度 | 传统RFM | 保险适配 | 说明 |
|------|---------|----------|------|
| R (Recency) | 最近购买时间 | 最近互动时间 / 最近保单生效时间 | 保险非高频消费，互动比购买更能反映活跃度 |
| F (Frequency) | 购买频次 | 保单持有数 / 咨询频次 / 理赔频次 | 多保单用户价值更高，咨询频次反映需求深度 |
| M (Monetary) | 消费金额 | 年缴保费总额 / 保额总额 / 家庭总保费 | 家庭总保费比个人保费更能反映客户价值 |

**保险RFM分层策略**：
- **高价值客户**（R近+F高+M高）：3个月内互动+持有3张以上保单+年缴保费>2万 → 权益运营+转介绍激励
- **潜力客户**（R近+F低+M中）：近期互动但保单少，保费中等 → 保障缺口诊断+家庭保单渗透
- **沉睡客户**（R远+F低+M低/中）：超6个月无互动 → 低压力唤醒+保单体检触达
- **流失风险客户**（R远+F高+M高）：多保单但长期不互动 → 优先关怀+续保提醒

### 2. 家庭生命周期与保险需求映射

```
家庭生命周期阶段 → 核心保险需求：

[单身期 22-28岁]
├── 医疗险（百万医疗基础保障）
├── 意外险（综合意外）
└── 重疾险（预算允许时配置，年轻保费低）

[家庭形成期 28-35岁 结婚→第一个孩子]
├── 医疗险（升级至中端医疗）
├── 重疾险（夫妻互保+保额充足）
├── 定期寿险（覆盖房贷+家庭责任）
├── 意外险（提升保额）
└── 少儿医疗/重疾（孩子出生后）

[家庭成长期 35-50岁 孩子教育阶段]
├── 重疾险（检视保额是否充足）
├── 寿险（根据负债变化调整）
├── 教育年金（子女大学教育金储备）
├── 医疗险（考虑升级高端医疗）
└── 家财险（自有住房配置）

[家庭成熟期 50-65岁 孩子独立]
├── 养老年金（退休收入补充）
├── 增额终身寿（资产传承）
├── 医疗险（重点关注意外医疗+住院）
├── 护理险（失能风险覆盖）
└── 防癌医疗险（替代常规医疗险的备选）

[退休期 65岁+]
├── 意外险（老年意外+骨折险）
├── 防癌医疗险（核保宽松）
├── 惠民保（兜底保障）
└── 长期护理险
```

### 3. 标签体系构建最佳实践

**五层标签架构**：

```
第一层：基础标签（事实型，不可变或缓慢变化）
├── 人口统计：年龄、性别、地域（省/市/区）
├── 社会经济：职业类别、收入区间、学历
├── 家庭结构：婚姻状况、子女数/年龄、父母赡养

第二层：保障标签（保单驱动，动态更新）
├── 保单清单：[{险种,保额,保费,生效日,到期日,缴费状态}]
├── 险种覆盖矩阵：{医疗:1,重疾:0,意外:1,寿险:0,年金:1,家财:0}
├── 保额汇总：{医疗保额,重疾保额,意外保额,寿险保额}
├── 保费汇总：{年缴总保费,月均保费}
└── 保单健康度：{即将到期:Y/N,存在缺口:Y/N,重复投保:Y/N}

第三层：健康标签（核保+医疗数据，敏感数据需脱敏）
├── 体检异常：{结节:甲状腺/乳腺/肺,三高:血压/血糖/血脂,BMI异常}
├── 既往病史：{疾病名称,确诊时间,治疗状态}
├── 核保结论：{标准体/次标准体/加费/除外/延期/拒保}
└── 健康风险等级：低/中/高（综合考虑年龄+异常指标数）

第四层：行为标签（交互驱动，实时更新）
├── 咨询偏好：{渠道:企微/小程序/电话,时段:早/中/晚,话题偏好}
├── 投保意向：高/中/低（基于浏览/咨询/对比行为）
├── 拒单原因：{健康告知/职业限制/价格/产品不合适}
├── 互动频次：{近30天互动次数,互动趋势:上升/平稳/下降}
└── 活跃等级：活跃(7天内)/一般(30天内)/低频(90天内)/沉睡(>90天)

第五层：需求标签（推断型，由上游标签+规则/模型推导）
├── 医疗刚需：基于健康状况+年龄+医疗险覆盖率
├── 大病保障：基于重疾险覆盖率+家庭病史+年龄
├── 家庭责任：基于家庭负债+子女数+寿险覆盖率
├── 养老储蓄：基于年龄+年金覆盖率+收入水平
├── 子女规划：基于子女年龄+教育金覆盖率
└── 出行保障：基于出行频次+意外险覆盖率
```

### 4. 标签元数据标准

每个标签需携带：
```json
{
  "tag_name": "医疗刚需等级",
  "tag_value": "高",
  "source": "rule_engine",
  "confidence": 0.85,
  "updated_at": "2024-01-15T10:30:00Z",
  "ttl_days": 90,
  "upstream_tags": ["健康风险等级", "医疗险覆盖率", "年龄"],
  "version": "v2.1"
}
```

## Instructions

1. **初始化画像**：采集用户基础标签→汇总保单生成保障标签→关联健康数据→追踪行为数据→推导需求标签
2. **增量更新**：仅更新变化标签，保留历史版本快照（用于趋势分析）
3. **标签置信度**：规则推导标签confidence=1.0，模型推断标签需标注confidence并定期校准
4. **隐私保护**：健康标签和收入标签需脱敏后输出，原始数据不出Agent边界
5. **输出格式**：统一使用JSON结构化输出，包含完整标签元数据
