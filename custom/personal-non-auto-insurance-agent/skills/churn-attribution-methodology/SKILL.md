# 保险用户流失归因方法论

## Purpose
为churn-recovery-agent提供系统化的流失归因分析框架，使其能够从行为数据+保单数据+画像数据中判定流失真正原因，并以因果推断方法论区分因果因素与相关因素，为召回策略选择提供可信证据。

## When to Use
- 用户保单到期未续/失效/退保时触发流失归因
- 构建流失预警模型需要标注训练数据时
- 评估不同召回策略的预期效果时
- A/B测试设计与结果分析时

## Core Knowledge

### 一、保险行业用户流失类型分类体系

#### 1.1 五类流失归因框架

```
┌──────────────────────────────────────────────────────────┐
│ Type 1: 价格敏感型流失 (Price-Sensitive Churn)             │
├──────────────────────────────────────────────────────────┤
│ 特征：用户认可保障价值，但保费超出心理承受范围               │
│ 行为信号：                                                │
│ ├── 历史保单均为低价位产品                                  │
│ ├── 多次浏览但未购买（价格犹豫）                            │
│ ├── 咨询中多次提及"太贵""超出预算"                          │
│ ├── 保费占收入比例较高（>10%）                             │
│ └── 有比价行为（多平台/多渠道询价）                         │
│ 因果机制：保费绝对值/相对值 → 支付意愿不足 → 放弃续保       │
│ 召回策略：优惠券/折扣 → 降低保额方案 → 分期缴费引导         │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│ Type 2: 产品不满型流失 (Product Dissatisfaction)           │
├──────────────────────────────────────────────────────────┤
│ 特征：对产品本身的保障范围/条款/服务不满意                   │
│ 行为信号：                                                │
│ ├── 保障与需求不匹配（买了不需要的）                         │
│ ├── 认为"买了不划算""从未理赔"                             │
│ ├── 觉得免赔额太高/报销比例太低                             │
│ ├── 发现更好的替代产品                                     │
│ └── 产品停售/条款变更通知后流失                             │
│ 因果机制：产品价值感知低 → 续保意愿低 → 流失                │
│ 召回策略：产品替换 → 保障缺口再触达 → 升级/降级引导         │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│ Type 3: 需求变化型流失 (Life-Stage Change)                 │
├──────────────────────────────────────────────────────────┤
│ 特征：用户生活状态变化，原有保障不再适用或优先级下降          │
│ 行为信号：                                                │
│ ├── 家庭结构变化（结婚/生育/子女独立）                       │
│ ├── 职业/收入大幅变化                                      │
│ ├── 居住地变更（迁移/出国）                                │
│ ├── 已通过其他渠道获得保障（如公司团险升级）                  │
│ └── 财务状况恶化（失业/降薪）                               │
│ 因果机制：需求变化 → 原有产品不再匹配 → 放弃续保             │
│ 召回策略：需求重新评估 → 产品重新匹配 → 灵活方案             │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│ Type 4: 遗忘/疏忽型流失 (Passive Churn)                    │
├──────────────────────────────────────────────────────────┤
│ 特征：用户有续保意愿但因遗忘/疏忽/流程繁琐而流失              │
│ 行为信号：                                                │
│ ├── 历史续保记录良好（已连续续保多年）                       │
│ ├── 最后互动时间在失效前（非长期沉默）                       │
│ ├── 未主动退保或表达不满                                   │
│ ├── 缴费方式为手动缴费（非自动扣款）                         │
│ └── 多保单用户（管理复杂度高导致遗漏）                       │
│ 因果机制：提醒不足/操作障碍 → 未及时缴费 → 被动流失          │
│ 召回策略：强化提醒 → 简化续保流程 → 开通自动扣款             │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│ Type 5: 理赔纠纷型流失 (Claims Dispute Churn)              │
├──────────────────────────────────────────────────────────┤
│ 特征：因理赔体验差（拒赔/少赔/慢赔）导致信任丧失             │
│ 行为信号：                                                │
│ ├── 近期有未结案或已结案理赔记录                            │
│ ├── 理赔结果为拒赔/部分赔付                                 │
│ ├── 理赔后有投诉/差评记录                                  │
│ ├── 理赔后互动频率骤降                                     │
│ └── 理赔金额与预期差距大                                   │
│ 因果机制：理赔体验差 → 信任崩塌 → 不再续保                  │
│ 召回策略：人工跟进 → 理赔复议 → 服务补救 → 关系修复         │
└──────────────────────────────────────────────────────────┘
```

#### 1.2 归因类型-关键特征矩阵

| 特征维度 | 价格敏感 | 产品不满 | 需求变化 | 遗忘疏忽 | 理赔纠纷 |
|---------|---------|---------|---------|---------|---------|
| 收入水平 | 低-中 | 不敏感 | 不敏感 | 不敏感 | 不敏感 |
| 保费/收入比 | 高(>10%) | 不敏感 | 不敏感 | 不敏感 | 不敏感 |
| 互动频次 | 正常 | 下降 | 波动 | 突然下降 | 骤降(理赔后) |
| 购买后时长 | 1-2年 | 2-3年 | 任意 | 多年 | 理赔后 |
| 理赔记录 | 少或无 | 可能有 | 无影响 | 无 | 有拒赔/少赔 |
| 已购保单数 | 少(1-2) | 中(2-3) | 中 | 多(3+) | 任意 |
| 退保/投诉 | 少 | 可能有 | 少 | 无 | 有 |
| 续保历史 | 中等 | 下降趋势 | 良好 | 良好 | 此前良好 |

### 二、续保决策行为学模型

#### 2.1 续保决策公式（行为经济学视角）

```
续保概率 = f(感知价值, 感知成本, 惯性, 触发事件)

├── 感知价值 = 保障需求强度 × 产品匹配度 × 信任度
│   ├── 保障需求强度：受年龄增长、健康状况、家庭责任驱动
│   ├── 产品匹配度：保障范围是否覆盖当前风险
│   └── 信任度：品牌信任 + 理赔体验 × 0.7 + 服务体验 × 0.3
│
├── 感知成本 = 保费绝对值 - 心理账户预算
│   ├── 保费绝对值：年缴保费
│   └── 心理账户预算：该用户为保障设定的心理支出上限
│
├── 惯性 = 续保年数 × 0.1 + 自动扣款(+0.3) + 保单托管(+0.1)
│   └── 每多续保一年，惯性增加约10%概率
│
└── 触发事件 = 理赔体验(+/-) + 产品停售(-0.4) + 竞品接触(-0.2)
    ├── 正面理赔：+0.2~0.4
    ├── 负面理赔：-0.5~0.8（理赔纠纷是最强负向触发）
    └── 产品停售：-0.4（强制性流失风险）
```

#### 2.2 续保率基准数据（行业参考）

| 险种 | 首年续保率 | 二年续保率 | 三年+续保率 | 续保率衰减特征 |
|------|----------|----------|-----------|-------------|
| 百万医疗险 | 55%-70% | 45%-60% | 40%-55% | 首年衰减最大，后趋于平稳 |
| 重疾险（长期） | 95%+ | 93%+ | 90%+ | 衰减极慢（长期险特征） |
| 意外险（短期） | 30%-50% | 25%-40% | 20%-35% | 续保率最低，遗忘型流失占比高 |
| 定期寿险（长期） | 97%+ | 95%+ | 93%+ | 衰减极慢 |
| 家财险 | 40%-60% | 35%-50% | 30%-45% | 中等 |
| 年金险 | 98%+ | 97%+ | 95%+ | 长期缴费，惯性极强 |

> 注：以上为行业估算参考值，实际续保率因产品/渠道/客群差异较大

### 三、流失归因因果推断框架

#### 3.1 区分因果因素 vs 相关因素

```
相关但非因果（易误判）：
├── 年轻用户流失率高 → 真正原因是收入低（价格敏感），非年龄本身
├── 多保单用户流失率低 → 真正原因是惯性+需求复杂度高，非保单数本身
├── APP活跃用户续保率高 → 真正原因是需求强+信任高驱动了活跃行为
└── 高端产品流失率低 → 真正原因是客户筛选（高收入+高需求），非产品类型

因果推断需要回答的反事实问题：
├── "如果该用户的保费降低20%，TA会续保吗？"（价格敏感型判定）
├── "如果该用户上次理赔没有被拒，TA会续保吗？"（理赔纠纷型判定）
├── "如果该用户收到了续保提醒短信，TA会续保吗？"（遗忘型判定）
└── "如果给该用户换上另一款产品，TA会续保吗？"（产品不满型判定）
```

#### 3.2 因果推断方法选择

| 归因场景 | 推荐因果方法 | 所需数据 |
|---------|------------|---------|
| 价格对续保的因果效应 | DML/IV | 保费、收入、续保结果，价格变动的工具变量 |
| 理赔体验对续保的因果效应 | PSM/DID | 理赔前后行为数据，理赔结果(赔/拒) |
| 提醒策略对续保的因果效应 | A/B Test/RCT | 实验组/对照组续保率 |
| 需求变化对续保的影响 | Panel Data FE | 用户时序数据（生活事件+续保） |
| 多因素联合归因 | DAG + DoWhy | 完整用户特征+行为+保单+理赔数据 |

### 四、召回策略匹配与A/B测试框架

#### 4.1 流失类型-召回策略映射

| 流失类型 | 首选策略 | 次选策略 | 不推荐策略 |
|---------|---------|---------|-----------|
| 价格敏感型 | 优惠/折扣 + 降额方案 | 分期缴费 + 比价引导 | 硬推销/忽视价格 |
| 产品不满型 | 产品替换 + 保障缺口触达 | 升级引导 + 需求重评 | 续保催促 |
| 需求变化型 | 需求重评 + 灵活方案 | 降级保留 + 转介绍 | 原产品续保催促 |
| 遗忘疏忽型 | 强化提醒 + 自动扣款 | 简化流程 + 短信/电话 | 产品变更 |
| 理赔纠纷型 | 人工跟进 + 关系修复 | 理赔复议 + 服务补救 | 自动化营销 |

#### 4.2 A/B测试设计模板

```
【实验设计】
├── 实验名称：{策略名称}召回效果测试
├── 目标指标：续保转化率（主要）/ 7天响应率（次要）
├── 实验周期：{开始日期}至{结束日期}（建议至少2周）
├── 样本量：每组至少500人（α=0.05, power=0.8, MDE=5%）
├── 分流比例：对照组50% / 实验组50%
├── 分流方式：用户ID哈希分流（保证一致性）

【策略配置】
├── 对照组：标准续保提醒（短信/T+0, T+7, T+14）
├── 实验组A：{策略A描述}
├── 实验组B：{策略B描述}（可选）

【评估指标】
├── 主要指标：续保转化率 = 续保成功数 / 目标用户数
├── 次要指标：响应率、响应时间、客单价变化
├── 异质性分析：按用户分层（年龄段/收入/流失类型）分别评估
└── 统计检验：卡方检验（转化率）+ t检验（连续指标）

【效果判定】
├── 策略显著优于对照（p<0.05）→ 推广
├── 策略无显著差异 → 保留观察或调整策略参数
├── 策略显著差于对照（p<0.05）→ 立即停止
└── 异质性显著 → 仅对有效子群推广
```

## Instructions

1. **流失检测触发**：保单到期未续/宽限期届满/退保申请 → 即时触发归因流程
2. **归因证据链**：每个归因判定至少包含3个支持信号，单信号不足以判定
3. **因果优先**：有条件时调用causal_layer区分因果与相关；无条件时基于规则+置信度标注
4. **策略个性化**：同一流失类型内，按用户画像微调策略参数（如优惠金额、产品推荐）
5. **闭环追踪**：召回结果反馈至归因模型，持续优化判定准确率
